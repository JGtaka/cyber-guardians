# 企画書:『サイバーガーディアンズ 〜セキュリティ勇者の冒険〜』(仮題)

改訂版 v3 — 2026-07確定の世界観(CLAUDE.md / docs/story.md)に整合。類似作品調査・技術選定確定・アートディレクション・要件・プロトタイプ実績を反映

## 1. コンセプト

**「戦うたびに、セキュリティに強くなる」**

ITセキュリティの知識を、ターン制RPGのバトルシステムに変換したミニRPG。
敵の攻撃は実在するサイバー攻撃、プレイヤーのスキルは実在するセキュリティ対策に1対1で対応しており、**遊び方を覚えること = セキュリティ対策を覚えること** になるのが最大の特徴。

### タイトル案
| 案 | 雰囲気 |
|---|---|
| サイバーガーディアンズ | 王道・チーム感(現仮題) |
| セキュリティクエスト | RPGらしさ重視 |
| FIREWALL SAGA | 海外風・かっこいい系 |
| まもれ!じょうほうしろ(情報城) | かわいい・教育系 |

### ターゲット
- メイン:中高生〜新社会人(セキュリティ研修の入口として)
- サブ:情報の授業・企業研修での利用

### 大会でのアピールポイント
1. **教育とゲーム性の融合**:攻撃と対策の対応関係が正確で、バトルの弱点システムに学習が埋め込まれている
2. **短時間で伝わる**:1バトル2〜3分。審査員のデモプレイに最適
3. **拡張性**:敵(=攻撃手法)はデータ追加だけで増やせる設計
4. **独自性**:類似作品調査済み。空白地帯を突いている(第2章)

## 2. 類似作品調査と差別化

「セキュリティ×ゲーム」の既存作は、アナログゲーム・クイズ型・TRPGに偏っており、
**「対策を弱点システムに落とし込んだ、一人でブラウザで遊べるターン制RPG」は前例が少ない**。

| 既存作品 | 形式 | 本作との違い |
|---|---|---|
| 警視庁 サイバーセキュリティボードゲーム | すごろく(印刷して遊ぶ) | アナログ・集合学習向け |
| JNSA「セキュリティ専門家 人狼」「Malware Containment」 | カード/ボードゲーム | 多人数前提。人狼はいらすとや採用の先行例 |
| Google「Space Shelter」「Interland」 | ブラウザのミニゲーム/クイズ | RPGバトルではない |
| IPA「ゲームで掴む!セキュリティのお仕事」 | 業務シミュレーション(約1時間) | 実務体験型。対象がやや上級 |
| Malware & Monsters | 協力型TRPG(マルウェア=モンスター) | 発想が最も近いが、人が集まる卓上ゲーム |
| Cryptomancer | ハッキング×魔法のTRPG | 同上 |
| バグハンター2 REBOOT(ふりーむ) | バグ退治RPG(7〜13時間) | 「IT知識×退治RPG」の先行例だが、題材は設計品質/バグでありセキュリティ対策ではない。書籍の副読本として物語で学ぶ長編 |

**差別化の要点(プレゼン用)**
- バグハンターが「IT知識×退治RPG」の面白さを証明。本作はその手法を**全員に必要なセキュリティ対策**に応用
- 学習の主戦場をストーリーではなく**バトルメカニクス(弱点=正しい対策)**に置く
- 20〜30分完結・ブラウザでURL一つ・ソロプレイ——既存作が埋めていない組み合わせ

## 3. 世界観・ストーリー(詳細は docs/story.md)

舞台は電子の国「**ネット王国**」。守る対象は2つ——王国の暮らしを支える**基幹システム**と、大切に保管された**王国民の情報**(カーネル王が預かる)。各章のボスは「システムを狙う攻撃」か「情報を狙う攻撃」のどちらかに分類できる。

プレイヤーは**王国の情報システム課に配属されたばかりの新米情シス**。
**主人公名はプレイヤー入力**(最大6文字・空白不可・デフォルト「シグ」)。入力画面で「ネットの世界では本名を使わない」の一言を添える(個人情報の教育ネタ)。
相棒はサポートAI「**クローンコード**」(愛称コード)。丁寧語(です・ます)で明るく健気、たまにぽんこつなオリジナルキャラクター。ともに各層を旅し、黒幕「**ゼロデイの魔王**」を目指す。

> クローンコードはバトル後に「それ、現実でも"あるある"なんですよ!」と解説する**ゲーム内の学習装置**。

**プロローグ「配属初日」**:合言葉「{n}1234」を登録する手元を小さな影が覗き見る(ショルダーハッキングの早期提示。影=ショルダーハック・ゴブリンは3章で再登場し伏線回収)。直後に魔王軍が侵攻。

**二部構成**:本編=つかう人編(全5章)/クリア後解放のEX章「かいはつしゃの塔」=つくる人編(XSS/SQLインジェクション/CSRF等、コードで防ぐ対策)。

### ステージ構成(章 = ネットワークの層)
| 章 | 舞台 | テーマ | ボス |
|---|---|---|---|
| 1 | メールの森 | フィッシング・スパム | フィッシング・アングラー |
| 2 | ウェブの街 | 偽サイト・マルウェア | トロイの木馬将軍 |
| 3 | パスワードの砦 | 認証・パスワード管理・覗き見対策 | ブルートフォース・ゴーレム(雑魚でショルダーハック・ゴブリン再登場) |
| 4 | 社内ネットワーク雪原 | ランサムウェア・ソーシャルエンジニアリング | ランサム・デーモン+ソーシャル・ウィッチ(裏切りの魔女) |
| 5 | クラウド天空城 | DDoS・ゼロデイ | DDoSドラゴン → ゼロデイの魔王 |

想定プレイ時間:20〜30分。章クリアごとに「セキュリティ手帳」(図鑑)へ攻撃と対策が記録される(この図鑑メカニクス自体が最終奥義「ミュートス」の伏線)。

## 4. バトルシステム(コア)

- ターン制コマンドバトル。コマンド:**たたかう / スキル / アイテム / ぼうぎょ**
- **弱点対策システム(最重要)**:敵ごとに正しい対策スキルが弱点
  - 正解 → WEAK POINT演出+大ダメージ
  - 不正解 → 微ダメージ+クローンコードがヒント(MPのムダで学ぶ)
  - 敗北 → クローンコードが弱点のヒントをくれて即リトライ(「負けて覚える」)
- ステータス読み替え:HP=**システム稼働率**、MP=**リソース**、攻撃=解析力、防御=堅牢性

※ 第1章プロトタイプ(reference/prototype.jsx)で名前入力/プロローグ/会話システム/雑魚戦+ボス戦/図鑑/最終決戦のスクリプトバトルを実装済み。
作者テストプレイで手応えを確認。バトルのテンポとイベントキュー方式は本実装でも踏襲する(移植時の見本)。

## 5. 敵キャラクター(攻撃手法との対応表)

| 敵キャラ | モチーフ | ゲーム内の行動 | 弱点スキル(=正しい対策) |
|---|---|---|---|
| ワームスライム | ワーム型マルウェア | 分裂して増える | ワクチンスキャン |
| フィッシング・アングラー | フィッシング詐欺 | 偽メール攻撃、リソースを奪う | URLかくにん |
| トロイの木馬将軍 | トロイの木馬 | 宝箱に化けて奇襲 | ダウンロードまえにスキャン |
| ブルートフォース・ゴーレム | 総当たり攻撃 | 毎ターン攻撃力上昇 | 二要素認証 |
| ショルダーハック・ゴブリン | ショルダーハッキング | 行動を先読み(プロローグの覗き見の正体。3章で再登場) | のぞきみブロック |
| ランサム・デーモン | ランサムウェア | アイテム欄を暗号化し身代金要求 | バックアップ |
| DDoSドラゴン | DDoS攻撃 | 多頭連続攻撃・行動遅延 | トラフィックせいぎょ |
| ソーシャル・ウィッチ | ソーシャルエンジニアリング | 味方のふりで毒バフ | かくにんのでんわ |
| ゼロデイの魔王 | ゼロデイ攻撃 | 未知の攻撃(既知の対策スキルが一切通じない) | **単独スキルでは倒せない。4段階の最終決戦でのみ撃破可能** |

**最終決戦(4段階・スクリプトバトル)**
1. 全スキル無効(ゼロデイの再現。一度絶望させる)
2. クローンコード覚醒「クローン展開!」——分身がファイアウォール/バックアップ/検知を同時展開(伏線回収:名前=多層防御)
3. 主人公の「緊急パッチ適用」で魔王に弱点が生まれる
4. 奥義「**ミュートス**」——魔王を解析・命名し手帳に記録して「未知を既知化」、完全消滅(現実対応:CVE採番・情報共有・パッチ配布)

> ラスボス設計 = 作品のメッセージ:「**銀の弾丸はない、多層防御が大事**」+「**記録と共有で、未知は既知になる**」

## 6. プレイヤーのスキル(=セキュリティ対策)

| スキル名 | ゲーム内効果 | 現実の対策 |
|---|---|---|
| ファイアウォール | 数ターン被ダメ半減 | ファイアウォール |
| ワクチンスキャン | マルウェア系特効 | ウイルス対策ソフト |
| きょうりょくパスワード | 防御アップ(パッシブ) | 長く複雑なパスワード |
| 二要素認証 | 認証系攻撃に特効/回避 | 2FA/MFA |
| あんごうか | 被ダメ一部無効 | データの暗号化 |
| バックアップ | 復活/アイテム欄復元 | 定期バックアップ(3-2-1) |
| パッチてきよう | 状態異常回復 | ソフトウェア更新 |
| URLかくにん | フィッシング特効 | リンク先・送り主の確認 |
| ログかんし | 敵の次の行動が見える | ログ監視・検知 |

## 7. アートディレクション(確定)

- **基調はドット絵**。ダークネイビー(#14142b系)+白枠4pxウィンドウのレトロRPG様式
- フォント:**DotGothic16**(Google Fonts)
- スプライト:ドット絵PNG+`image-rendering: pixelated`。仮素材はbox-shadowドット絵で代用可(プロトタイプで実証済み)
- **会話・図鑑の解説イラストは「いらすとや」**(JNSA人狼で採用実績あり)
- **絵柄の混在は設計として分離**:ゲーム世界=ドット絵/現実の解説パート=いらすとや。
  切り替わり自体が「ここからは現実の話」の合図として機能する
- いらすとやの**使用点数をASSETS.mdで管理**(商用は21点以上で有償規定。大会規約の第三者素材条項も確認)
- クレジット表記(素材出所)をREADME・ゲーム内に明記

## 9. 要件

### 機能要件(コア以外)
- タイトル画面/章選択/図鑑/画面遷移設計
- 名前入力画面(最大6文字・空白不可・デフォルト「シグ」。台詞は {n} プレースホルダで差し込み)
- 1戦目=チュートリアル戦(クローンコードが操作と弱点システムを誘導)
- ゲームオーバー=学びに変換(ヒント+即リトライ)
- 会話パートに既読スキップ/早送り(リトライ周回とテンポ対策)
- デバッグメニュー:章ジャンプ・セーブ初期化・ミュート(審査ブース対策。公開ビルドでは隠す)

### 非機能要件
- 初回ロード3秒以内(音声・画像は遅延読み込み)
- スマホ幅(375px)対応。タップ操作前提
- アクセシビリティ:色だけに情報を持たせない(数値併記)、11px未満禁止、キーボード操作
- 安全性:dangerouslySetInnerHTML禁止、ユーザー入力は必ずエスケープ

### 教育品質(最重要)
- 解説はIPA・総務省等の一次情報を典拠とし、出典を図鑑・READMEに明記
- 「これで完全に防げる」等の言い切り禁止(多層防御の設計と矛盾させない)
- 攻撃の再現手順は書かない。学ばせるのは「見分け方と守り方」

### 大会対応
- 規約確認:第三者素材の可否・提出形式・発表時間・審査基準(評価軸にプレゼンを対応付ける)
- 提出物:README(概要/遊び方/技術構成/出典/クレジット)、スクリーンショット、30秒プレイ動画
- デモ保険:会場ネット断に備えローカルでも起動できる状態を用意

## 10. ロードマップとリスク管理

| フェーズ | 内容 | 状態 |
|---|---|---|
| プロトタイプ | 第1章一式:名前入力/プロローグ/会話システム/雑魚戦+ボス戦/げんじつファイル付き図鑑/最終決戦スクリプトバトル(reference/prototype.jsx) | **完了** |
| v3 | Vite+TSへ移植 → localStorageセーブ+タイトル/章選択/図鑑画面+SE4種+デバッグメニュー → 第1章の完成度を上げる | **大会提出ライン(次)** |
| v4 | 第2〜4章(トロイの木馬将軍/ゴーレム+ゴブリン再登場/ランサム・デーモン+魔女) | 後 |
| v5 | 最終章解禁(ミュートス)→ EX章「かいはつしゃの塔」 | 後 |

- **最終章の公開方針**:最終決戦は2〜4章の実装が終わるまで公開ビルドに導線を出さない(積み重ねなしで見せると興ざめ)。締切が厳しい場合は提出版=1〜2章+「最終決戦封印中」の予告演出とし、プレゼンで審査員にのみ最終決戦をデモする
- **スコープ死守**:「1章完成」最優先。章追加はデータ追加で拡張
- **第三者テストプレイ必須**:作者は弱点を知っているため初見の試行錯誤の楽しさを測れない
- **提出1週間前=機能追加禁止日**。以降はバグ修正と磨き込みのみ
- バランス数値の変更は理由をコミットメッセージに残す

## 11. 発表(プレゼン)で推すポイント

1. デモ:わざと間違った対策→「効かない」→正しい対策で撃破、の流れで教育設計を一目で伝える
2. 敵=攻撃/スキル=対策の対応表(5・6章)をスライド1枚で
3. 類似作品調査(2章)から「空白地帯」を示し独自性を語る
4. ラスボス戦の「クローン展開=多層防御」「ミュートス=記録・共有で未知を既知化」を作品のメッセージとして締める(公開版で封印中なら審査員限定デモ)
