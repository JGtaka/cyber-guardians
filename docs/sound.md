# サウンド設計書 — サイバーガーディアンズ

BGM・SEの選定基準と実装仕様。素材ファイルの入手・配置は人間の作業、
実装(src/game/sound.ts)はClaude Codeの作業。

## 方針

- **SEが先、BGMは後**(工数小・効果大の順)
- 全音声にミュートボタン必須(タイトル+バトル画面。状態はセーブデータに含める)
- ブラウザの自動再生制限のため、BGM開始は必ず最初のユーザー操作(タイトルの決定)に紐づける
- 素材の出所は ASSETS.md に記録し、README にクレジット表記

## SEリスト(トリガー対応表)

### 優先度S(これだけで体感が激変する4種)
| SE | トリガー | 音のイメージ |
|---|---|---|
| 決定 | コマンド選択・ボタン押下 | レトロなピコッ(短い) |
| メッセージ送り | ▼クリック | 控えめなピッ(連打されるので短く小さく) |
| 攻撃ヒット | 通常攻撃・スキル命中 | 打撃のザシュ/バシッ |
| **WEAK POINT** | 弱点ヒット演出 | 派手なキュイーン+クリティカル音。**最重要**。画面フラッシュと同時に鳴らす |

### 優先度A(バトルの手触り)
| SE | トリガー | 音のイメージ |
|---|---|---|
| 被弾 | プレイヤーがダメージ(画面シェイクと同時) | ドスッ(低め) |
| 効いていない | 弱点以外のスキル/魔王無効化 | スカッ/ポフッ(気の抜けた音) |
| バフ展開 | ファイアウォール展開 | シールドのシュイン |
| 回復 | リソース回復・章間回復 | キラキラ上昇音 |
| 敵撃破 | 敵消滅 | レトロな消滅音(ピシューン) |
| 手帳記録 | 「きろくされた!」表示 | ページをめくる/スタンプ音 |

### 優先度B(演出強化。v3後半〜v4)
| SE | トリガー |
|---|---|
| 警報 | プロローグの侵攻イベント(ビーッ!) |
| 覗き見・気配 | ショルダーハッキング登場(ピキーン/ゾクッ) |
| ゲームオーバー | システムダウン時(下降音) |
| クローン展開 | 最終決戦・覚醒(v5。多重に重なる展開音) |
| ミュートス | 最終奥義(v5。荘厳な決め音+記録スタンプ) |

|---|---|---|---|

## 実装仕様(src/game/sound.ts)

- Howler.js を使用。SEは起動時プリロード、BGMは遅延ロード(非機能要件:初回3秒)
- API 案:`playSe(name)` / `playBgm(name)`(クロスフェード0.5s)/ `stopBgm()` / `toggleMute()`
- 音量初期値:BGM 0.4 / SE 0.7(SEが主役。テストプレイで調整)
- ミュート状態は localStorage セーブに含める
- ファイル配置:`public/bgm/`・`public/se/`。ファイル名は用途名(btl_normal.mp3, se_weak.mp3 等)
  ※素材の元ファイル名のままにしない(出所はASSETS.mdで管理)

## 選定の進め方(人間の作業)

1. 効果音ラボで優先度SのSE4種を選ぶ(「戦闘」カテゴリに定番が揃っている)
2. 魔王魂でバトル2曲(通常/ボス)を選ぶ。**候補を各2〜3曲落として、実際のゲーム画面を
   動かしながら鳴らして決める**(音単体で聴くのと画面と合わせるのとで印象が変わるため)
3. ファイルをリネームして public/ に配置 → ASSETS.md 記録 → Claude Code に実装依頼
