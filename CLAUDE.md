# CLAUDE.md — サイバーガーディアンズ 〜セキュリティ勇者の冒険〜(仮題)

アプリ大会向けの、ITセキュリティ学習ターン制RPG。設計・技術選定は確定済み。
このファイルが唯一の正であり、以下の決定を変更する場合は必ずユーザーに確認すること。
詳細仕様:@docs/design.md(企画書)/ @docs/story.md(ストーリー設計・確定版)

## プロジェクト概要

- コンセプト:「戦うたびに、セキュリティに強くなる」
- 敵 = 実在のサイバー攻撃、スキル = 実在の対策。**弱点対策システム**が核:
  正解スキル → WEAK POINT演出+大ダメージ / 不正解 → 微ダメージ+相棒がヒント
- 二部構成:本編=つかう人編(全5章)/ EX章「かいはつしゃの塔」=つくる人編(XSS/SQLi/CSRF)
- 想定プレイ時間 20〜30分

## 世界観・キャラクター(2026-07 確定版)

- 舞台:**ネット王国**。守る対象は2つ——**基幹システム**(王国の運用)と**王国民の情報**。
  各章のボスは「システムを狙う攻撃」か「情報を狙う攻撃」に分類できる
- 主人公:**電子王国・情報システム課に配属された新米情シス**。
  **名前はプレイヤー入力**(最大6文字・空白不可・デフォルト「シグ」)。
  入力画面で「ネットでは本名を使わない」の一言を添える。
  台詞・テキストは {n} プレースホルダで名前を差し込む方式
- 相棒:サポートAI「**クローンコード**」(愛称コード)。丁寧語(です・ます)、明るく健気、
  たまにぽんこつ。既存キャラの固有要素の模倣は禁止(オリジナルキャラとして扱う)
- ラスボス:ゼロデイの魔王。最終決戦は4段階——全スキル無効 → クローン展開(分身で
  多層防御=名前の伏線回収) → 緊急パッチ適用(弱点生成) → 奥義**ミュートス**
  (解析・命名・手帳記録で「未知を既知化」=図鑑メカニクスが伏線。現実対応:CVE・情報共有)
- プロローグ:配属初日。合言葉「{n}1234」の覗き見(ショルダーハッキング早期提示、
  3章でゴブリンとして回収)→ 魔王軍侵攻

## 最終章の公開方針(重要)

- **最終決戦は2〜4章の実装が終わるまで公開ビルドに導線を出さない**(積み重ねなしで
  見せると興ざめのため)。reference/prototype_ch1.jsx に動作確認済みの実装
  (スクリプトバトル4段階)があるが、これは移植時の見本。公開版でボタンを出すのはv5から
- 締切が厳しい場合の代替案:提出版は1〜2章+「最終決戦封印中」の予告演出とし、
  プレゼンで審査員にのみ最終決戦をデモする

## 技術スタック(確定・変更禁止)

- Vite + React + TypeScript / useReducer(バトル=状態遷移マシン)/ Tailwind CSS
- サウンド:Howler.js+フリー素材(SE4種を優先:攻撃ヒット/WEAK POINT/メッセージ送り/決定。
  BGM開始は最初のクリックに紐づけ、ミュートボタン必須)
- ゲームデータは src/data/ に分離(enemies / skills / story / zukan)。データ駆動を厳守
- セーブ:**localStorage のみ**(キー "cyber_guardians_save"。名前・章・図鑑解放を含む)。
  DB・サーバー・Supabase は使わない
- デプロイ:Vercel。main = 本番 / dev = 作業。main へ直接 push しない
- 開発環境:WSL2内リポジトリ+Docker(node:22-slim)。
  vite.config.ts に server: { host: true, watch: { usePolling: true } }

## アートディレクション

- ドット絵基調。ダークネイビー #14142b+白枠4pxウィンドウ。フォント DotGothic16
- スプライト:PNG+`image-rendering: pixelated`(仮素材はbox-shadowドット絵で可)
- 会話・図鑑の解説イラストは「いらすとや」(使用点数をASSETS.mdで管理。商用21点以上有償)
- 絵柄の分離:ゲーム世界=ドット絵/現実解説パート=いらすとや

## 教育コンテンツの品質ルール(最重要)

- 解説はIPA・総務省・JPCERT/CC等の一次情報を典拠。図鑑の出典は**タップで開くリンク**
  (恒久性の高い公式ページ優先、target="_blank" + rel="noopener noreferrer" 必須)
- 実名ルール:攻撃名・用語・公的機関は実名OK/被害組織・個人は必ず属性表記でフィクション化。
  「〜の事例が報告されている」調。揶揄・過度な恐怖表現禁止
- 「完全に防げる」等の言い切り禁止。攻撃の再現手順は書かない
- 提出前チェックに「全出典リンクの生存確認」を含める

## 非機能要件

- 初回ロード3秒以内 / スマホ幅375px対応(タップ操作前提)
- dangerouslySetInnerHTML禁止。名前入力を含むユーザー入力は必ずエスケープ(Reactの標準機構を使う)
- 色だけに情報を持たせない(数値併記)。フォント11px未満禁止
- 会話パートに既読スキップ/早送りを実装(リトライ周回とテンポ対策)
- デバッグメニュー:章ジャンプ・セーブ初期化・ミュート(公開ビルドでは隠す)
