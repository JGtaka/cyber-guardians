# CLAUDE.md — サイバーガーディアンズ 〜セキュリティ勇者の冒険〜(仮題)

アプリ大会向けの、ITセキュリティ学習ターン制RPG。設計・技術選定はチャットで確定済み。
このファイルが唯一の正であり、以下の決定を変更する場合は必ずユーザーに確認すること。

## プロジェクト概要

- コンセプト:「戦うたびに、セキュリティに強くなる」
- 敵 = 実在のサイバー攻撃(ワーム、フィッシング、ブルートフォース等)、
  プレイヤーのスキル = 実在の対策(ワクチンスキャン、URL確認、二要素認証、ファイアウォール等)
- コアメカニクス:**弱点対策システム**。敵ごとに正しい対策スキルが弱点。
  正解 → WEAK POINT演出+大ダメージ / 不正解 → 微ダメージ+相棒パッチがヒント
- 相棒AI妖精「パッチ」が解説役。撃破後に「セキュリティ手帳」(図鑑)で現実の攻撃と対策を解説
- ラスボス「ゼロデイの魔王」は単一スキルでは倒せず、多層防御の組み合わせでのみ勝てる設計
  (作品メッセージ:「銀の弾丸はない、多層防御が大事」)
- 想定プレイ時間 20〜30分。全5章(メールの森→ウェブの街→パスワードの砦→社内NW雪原→クラウド天空城)
- 詳細仕様は @docs/design.md を参照

## 技術スタック(確定・変更禁止)

- Vite + React + TypeScript / 状態管理は useReducer(バトルは状態遷移マシンとして実装)
- スタイル: Tailwind CSS。サウンド: Howler.js(実装は後回しで良い)
- ゲームデータ(敵・スキル)は src/data/ の JSON/TS データとして分離。データ駆動を厳守
- セーブ: **localStorage のみ。DB・サーバー・Supabase は使わない**(静的サイトとして完結)
- デプロイ: Vercel。ブランチ運用: main = 本番(審査員用URL) / dev = 日常作業。main へ直接 push しない
- 開発環境は Docker(node:22-slim + compose)。Windows ホストのため
  vite.config.ts に server: { host: true, watch: { usePolling: true } } を設定

## アートディレクション

- ドット絵基調。ダークネイビー(#14142b 系)+白枠4pxウィンドウ(ドラクエ風)
- フォント: DotGothic16(Google Fonts)
- スプライトはドット絵PNG + `image-rendering: pixelated`。プレースホルダは box-shadow ドット絵で可
- 会話パート・図鑑の解説イラストは「いらすとや」を使用。
  **使用点数を必ず ASSETS.md に記録**(規約: 商用は21点以上で有償。大会規約も要確認)
- 絵柄の混在は設計として分離: ゲーム世界=ドット絵 / 現実の解説パート=いらすとや

## 教育コンテンツの品質ルール(最重要)

- セキュリティ解説は IPA・総務省等の一次情報を典拠とし、出典を図鑑データに持たせる
- 「これで完全に防げる」等の言い切り表現は禁止。多層防御の設計と矛盾させない
- 攻撃の再現手順は書かない。学ばせるのは「見分け方と守り方」のみ

## 非機能要件

- 初回ロード3秒以内。音声・画像は遅延読み込み
- スマホ幅(375px)でレイアウトが崩れないこと。コマンドはタップ操作前提
- dangerouslySetInnerHTML 使用禁止。ユーザー入力は必ずエスケープ
- HPバー等、色だけに情報を持たせない(数値併記)。フォントサイズ11px未満禁止

## ロードマップと現状

  ユーザーのテストプレイ済みで「面白い」評価。バトルのテンポとイベントキュー方式はこれを踏襲する
